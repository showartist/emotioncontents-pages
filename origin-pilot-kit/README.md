# ORIGIN PILOT KIT · 검증엔진

GitHub Pages에 바로 올릴 수 있는 **정적 검증 운영 웹앱**입니다.

## 이 버전에서 실제로 되는 것
이제 이 앱은 단순 생성기가 아니라, 아래 흐름을 **실제로 운영**할 수 있습니다.

1. 운영자가 IP 브리프를 입력한다.
2. 앱이 **검증 가설 / 실험 설계 / 판단 기준**을 만든다.
3. 운영자가 **응답자용 테스트 링크**를 생성해 외부로 보낸다.
4. 응답자는 링크에서 자극을 보고 답한 뒤 **응답 패키지**를 만든다.
5. 운영자는 돌아온 패키지를 붙여넣어 **실제 증거**로 쌓는다.
6. 대시보드가 **GO / ITERATE / PIVOT / STOP** 판단을 업데이트한다.

즉, 핵심은 “5개 생성”이 아니라 **사람에게 물어볼 수 있는 검증 상태로 바꾸는 것**입니다.

## 포함 기능
- 사주보이즈 / 공연형 샘플 자동 로드
- IP 브리프 입력
- 검증 가설 자동 생성
- 실험 설계 자동 생성
- **응답자 테스트 링크 생성**
- **응답자 전용 모드(`?run=` 링크)**
- **응답 패키지 문자열 생성 / 복사 / JSON 저장**
- 운영자 측 응답 패키지 흡수
- 수동 응답 추가
- 지표 집계
  - 핵심 인식 적중률
  - 재방문 YES 비율
  - 차별감 평균
  - 명확성 평균
  - 행동 전환 신호
- 현재 판단 자동 계산
- Markdown 리포트 저장
- 프로젝트 JSON 내보내기 / 불러오기
- localStorage 자동 저장

## 테스트 운영 방법
### 1) 운영자 모드
기본 접속:

```text
https://showartist.github.io/emotioncontents-pages/origin-pilot-kit/
```

여기서 브리프를 입력하고 **검증 프로젝트 생성** → **응답자 링크 생성** 순서로 진행합니다.

### 2) 응답자 모드
운영자가 만든 링크는 이런 형태입니다.

```text
https://showartist.github.io/emotioncontents-pages/origin-pilot-kit/?run=...
```

응답자는 이 링크에서 자극을 보고 답한 뒤, **응답 패키지**를 복사하거나 JSON으로 저장합니다.

### 3) 증거 흡수
운영자는 **증거 수집** 탭에서:
- 응답 패키지 문자열 붙여넣기
- 또는 응답 JSON 불러오기

방식으로 실제 응답을 모읍니다.

## 기술 제약
- 백엔드 없음
- 외부 API 없음
- GitHub Pages 호환 정적 구조
- 응답은 중앙 서버에 자동 저장되지 않음
- 외부 공유는 **링크 + 응답 패키지 회수 방식**으로 동작

## 왜 이 구조인가
해커톤/캡스톤 초기에는 “AI가 IP를 성공 판정한다”는 주장보다,
**AI가 검증 가능한 실험과 실제 증거 수집 흐름을 만들어준다**는 구조가 더 현실적이고 설득력 있습니다.
